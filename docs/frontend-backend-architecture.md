# 前后端分离与轻量部署

## 运行结构

- `resume_ui/` 是独立 Vue 3 前端工程。开发时由 Vite 运行，生产构建输出到 `static/app/`。
- `boss_app.py` 只负责 JSON API、WebSocket、认证、SQLite 与浏览器自动化。
- 生产环境建议保持同源：FastAPI 同时托管构建后的静态文件和 API。这样不需要额外网关，也不会暴露跨域凭据。
- 开发环境使用 Vite 的 `/api` 与 `/ws` 代理通过 HTTPS/WSS 连接 `127.0.0.1:8010`，并仅对本地自签名证书关闭代理校验。

## 模块边界

| 前端负责 | 后端负责 |
| --- | --- |
| 页面路由、表单即时校验、加载与错误状态 | 多用户认证、角色权限、密码哈希、会话签名与登录限流 |
| 搜索条件编辑、分页位置、当前会话等临时状态 | 每日上限、公司去重、岗位过滤与自动投递安全规则 |
| 10 秒短缓存和可见页面刷新 | 5 秒聚合缓存、SQLite 查询与数据一致性 |
| 收到 WebSocket 事件后按需刷新 | BOSS 浏览器控制、消息同步、计划调度和事件广播 |
| 简历编辑交互、实时预览和模块排序 | 简历解析、真实性校验、DOCX/PDF 生成和持久化 |
| 不保存 AI Key，仅提交新值 | AI Key 明文只保留在服务器设置中，API 只返回是否已配置 |

业务规则不能只放在前端。前端校验用于体验，后端校验用于安全和一致性。

## 低压力策略

1. 概览只请求 `/api/dashboard/summary`，后端缓存 5 秒，前端缓存 10 秒。
2. 有 WebSocket 事件时才主动刷新；兜底轮询为 15 秒，并在页面不可见时停止。
3. 聊天不持续控制 BOSS 浏览器。普通消息接口只读本地缓存，用户点击同步时才访问 BOSS 页面。
4. 岗位列表默认每页 20 条，后端单次最多返回 100 条。
5. 统计使用 SQL `COUNT`，不再为了显示数字读取一万条岗位记录。
6. 简历编辑器和各业务模块使用动态导入，登录页不会加载完整简历编辑器。
7. 会话令牌由 HMAC 签名并使用内存会话缓存，不需要每个 API 请求查询数据库。
8. 在线心跳每 60 秒发送一次，后端每个会话约 50 秒才落库一次；管理员统计只读聚合数据。

## 登录模型

用户首页位于 `/`，用户登录后的工作台位于 `/app`，管理员后台位于 `/MyJobaAdmin`。账号、会话和活跃统计保存在 `.boss_profile/auth.db`：

- 密码使用 PBKDF2-SHA256 加盐哈希，不保存明文。
- 登录会话写入 `HttpOnly`、`SameSite=Lax` Cookie。
- 连续失败登录会触发内存限流。
- 普通用户只能在用户端注册，公开注册接口永远不会授予管理员角色。
- 新认证库会创建默认超级管理员 `Admin / 123456*`，首次登录必须修改密码。
- 管理员账号只能由超级管理员在专用后台创建，新管理员同样需要首次改密。
- 修改密码、退出工作台或停用账号会使对应旧会话失效。
- 除健康检查、登录接口和静态文件外，全部业务 API 与 WebSocket 都需要认证。

管理员后台展示普通用户注册数、当前在线、今日活跃、累计在线时长、累计登录次数、近 7 日趋势和最近会话。管理员可以开关普通用户注册并启停普通账号；超级管理员还可以创建和管理管理员账号。统计中的 IP 和 User-Agent 只保存不可逆短哈希，不保存原文。

工作台账号只保护 MyJob。招聘平台账号仍在独立浏览器中完成登录，两个登录状态不会混淆。

## 部署建议

轻量服务器使用单进程即可，因为浏览器自动化本身要求单实例：

```bash
cd resume_ui
npm install
npm run build
cd ..
python boss_app.py --host 127.0.0.1 --port 8010
```

该命令默认使用 HTTPS，缺少证书时会在 `.boss_profile/tls/` 自动生成只适合本机开发的自签名证书。也可以直接指定受信任证书：

```bash
python boss_app.py --ssl-certfile /path/fullchain.pem --ssl-keyfile /path/privkey.pem
```

等价环境变量为 `MYJOB_TLS_CERT` 和 `MYJOB_TLS_KEY`。需要公网访问时，也可以在 FastAPI 前放置 Caddy 或 Nginx 终止 HTTPS，然后设置：

```text
MYJOB_SECURE_COOKIE=true
MYJOB_CORS_ORIGINS=https://你的域名
MYJOB_SESSION_HOURS=12
```

若反向代理到本机 HTTP，上游启动命令需显式添加 `--http`，公网入口仍必须为 HTTPS。

不要启动多个 Uvicorn worker。多个 worker 会各自持有浏览器、调度器和内存缓存，既浪费资源也会产生冲突。
