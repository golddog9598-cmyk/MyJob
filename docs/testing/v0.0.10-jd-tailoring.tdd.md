# V0.0.10 JD tailoring TDD evidence

## Source and journeys

The journeys were derived from the approved V0.0.10 change request; no external plan file was used.

- As a user, I can configure an independent daily application limit for each recruitment platform, never above 50.
- As a user, I am stopped before a platform action when that platform reaches its daily limit.
- As a user, I can tailor only the five approved resume modules against a locally cached JD.
- As a user, I can choose low, medium, high, or extreme fact constraints and must confirm proposed facts or metrics before applying them.
- As a user, my JD and AI credential do not pass through the MyJob backend.

## Known gaps

- Live AI output quality depends on the user-selected provider and model; automated tests mock the external service and do not send credentials.
- Recruitment sites can change their DOM. Full-JD extraction fails closed and asks the user to retry from a valid job-detail page.
- Browser CORS policy may reject a provider that does not allow direct browser calls; MyJob deliberately does not proxy the request through its backend.

Checkpoint commits were deferred because the orchestration workflow requires explicit pre-commit approval. RED/GREEN evidence is preserved above and in the test files.
